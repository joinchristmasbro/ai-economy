# `_brain/` — Cérebro do Workspace

Memória viva do sistema. Tudo que você captura, lê, escuta sobre o nicho, mercado e clientes vira nota estruturada aqui. Quanto mais você usa, mais inteligente fica.

## Estrutura

```
_brain/
├── inbox/           ← drop zone (captures rápidos)
├── conceitos/       ← evergreen notes (modelos mentais, frameworks)
├── fontes/
│   ├── youtube/     ← vídeos extraídos
│   ├── reddit/      ← posts/threads
│   ├── ads/         ← criativos analisados
│   └── artigos/     ← textos/blog posts
├── insights/        ← weekly briefs + cross-source patterns
└── raw/             ← outputs brutos de pesquisa multi-agent
```

## Fluxo de uso

```
captura rápida          → inbox/                  (qualquer hora)
processamento           → /sintetizar              (semanal)
extração estruturada    → /knowledge-extractor    (de fonte específica)
síntese semanal         → /intel-synthesis        (sextas)
busca de outliers       → /youtube-intel niche-scan (mensal)
```

## Princípios

1. **Captura primeiro, organiza depois.** Inbox é zona de drop livre — `/sintetizar` distribui depois.
2. **Estágios de confiança.** Toda nota tem `stage`: `seedling` (recém-capturada) → `budding` (revisitada) → `evergreen` (consolidada, citada por 3+ outras notas).
3. **Concept-oriented**, não source-oriented. Padrão recorrente em 3+ fontes vira conceito em `conceitos/`, com links pras fontes originais.
4. **Linked thinking.** Toda nota tem frontmatter com `linked-clients`, `linked-concepts`, `linked-sources` — cria graph navegável.
