# Perfil: Técnico com skill, sem negócio

Aluno tem skill técnica em IA/dev/automação mas não monetizou ou não tem nicho definido. Setup foca em **descobrir mercado primeiro**, depois aplicar skill como diferencial competitivo.

## Linguagem default

- "Pra quem você vende?" → "qual mercado/nicho você ataca?"
- "Cliente" → ainda hipotético no início, vira concreto após mapear nicho
- "Skill atual" → ferramenta/stack que já domina (n8n, Make, OpenAI API, agentes, GHL, Synthflow)

## Foco primário sugerido (quase sempre nessa ordem)

1. **Aprofundar nicho** — sem nicho específico, skill técnica vira commodity. Definir mercado é movimento #1.
2. **Spy-ads** — paralelo ao mapeamento, ver o que já tá rodando no nicho.
3. **Estruturar oferta** — só faz sentido depois de nicho mapeado (com dores reais).
4. **Validar com copiloto** — fechamento.

## Anti-padrões frequentes

- "Vou montar um agente de IA genérico e depois penso pra quem vendo" → ordem invertida, agente sem nicho não vende
- "Vou aceitar trabalhos por hora pra começar a faturar" → entra no modelo errado, difícil sair
- "Meu skill é o diferencial, vou vender a stack" → cliente compra outcome, não stack
- "Vou cobrar barato pra fechar primeiro cliente" → race-to-bottom, ancora pricing baixo

## Skills mais usadas

`/mapear-mercado` (CRÍTICO) · `/escanear-concorrencia` · depois pipeline normal

`/copiloto` é útil pra refinar pitch — perfis técnicos tendem a explicar a IA, não o outcome.
